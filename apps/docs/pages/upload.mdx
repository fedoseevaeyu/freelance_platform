# Загрузка файлов

Вы узнаете как будет устроена загрузки контента, такого как изображения, видео, документы и т.д., на сервер.

## Загрузка одного файла

Сделайте запрос `multipart/form-data` к маршруту `/upload` с файлом, который будет загружен, в поле `file`.

<Callout type="warning">
  <strong>Предупреждение:</strong> Ограничение по размеру файла составляет 10 МБ.
</Callout>

<Callout type="warning">
  <strong>Предупреждение:</strong> Необходимо указать заголовок авторизации
</Callout>

Необязательные параметры запроса:

- `to` - Папка, в которую файл должен быть загружен в supabase. Если не указано иное, файл будет загружен в папку `images`.

Ответ

```json
{
  "path": "assets/a-long-random-string-randomUUID-filename.extension"
}
```

Чтобы использовать загруженный файл, вы можете использовать `path`, возвращаемый в файле, и префиксить его переменной среды `STORAGE_BUCKET_URL`.

## Загрузка нескольких файлов

Сделайте запрос `multipart/form-data` к маршруту `/upload` с файлами, который будут загружены, в поле `files`.

<Callout type="warning">
    <strong>Предупреждение:</strong> Ограничение по размеру файла составляет 10 МБ.
</Callout>

<Callout type="warning">
    <strong>Предупреждение:</strong> Необходимо указать заголовок авторизации
</Callout>

Необязательные параметры запроса:

- `to` - Папка, в которую файл должен быть загружен в supabase. Если не указано иное, файл будет загружен в папку `images`.

Ответ

```json
{
  "paths": ["url1.jpg", "url2.jpg"]
}
```
